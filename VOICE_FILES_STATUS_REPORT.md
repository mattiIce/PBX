## Solution: How to Commit Voice Files

If you want to commit the voice files to the repository, follow these steps:

### Step 1: Generate the Voice Files

```bash
cd /home/runner/work/PBX/PBX

# Option A: Using Google TTS (requires internet, best quality)
pip3 install gTTS pydub
python3 scripts/generate_tts_prompts.py --company "Aluminum Blanking Company"

# Option B: Using eSpeak (offline, robotic voice)
python3 scripts/generate_espeak_voices.py --company "Aluminum Blanking Company"

# Option C: Using Festival (offline, better quality)
sudo apt-get install festival festvox-us-slt-hts
python3 scripts/generate_natural_voices.py --engine festival --company "Aluminum Blanking Company"
```

### Step 2: Update .gitignore to Track Voice Files

Remove or comment out these lines from `.gitignore`:
```
# Generated voice files (generate with scripts/generate_tts_prompts.py)
auto_attendant/*.wav
voicemail_prompts/*.wav
```

Or create an exception for specific files while keeping the pattern:
```bash
# Add voice files to git
git add -f auto_attendant/*.wav
git add -f voicemail_prompts/*.wav
```

### Step 3: Commit and Push

```bash
# Verify the files are ready to commit
git status

# Commit the voice files
git add auto_attendant/*.wav voicemail_prompts/*.wav
git commit -m "Add pre-generated voice prompts for auto attendant and voicemail"

# Push to remote
git push origin main
```

## Verification Commands

### Check if voice files exist locally:
```bash

ls -lh voicemail_prompts/*.wav 2>/dev/null || echo "No .wav files in voicemail_prompts/"
```

### Check if voice files are tracked in git:
```bash
git ls-files auto_attendant/*.wav voicemail_prompts/*.wav
```

### Generate and verify voice files:
```bash

python3 scripts/generate_tts_prompts.py --company "Your Company"

ls -1 auto_attendant/*.wav voicemail_prompts/*.wav | wc -l

```

## Recommendation

**I recommend keeping the current approach (Option 1)** where voice files are generated by users because:

1. **Customization**: Each organization can generate files with their own company name
2. **Repository Size**: Keeps the repo smaller and faster to clone
3. **Best Practice**: Generated files should typically not be in version control
4. **Flexibility**: Users can choose between Google TTS, Festival, or eSpeak based on their needs

However, if you prefer convenience for users (so they don't need to generate files), Option 2 is also valid.
